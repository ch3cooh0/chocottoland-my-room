# 冒険者のお部屋

チョコットランドの冒険者のお部屋です。

装備の組み合わせやドピ計算ができるソフトウェアを目指して開発しています。

## できること

まだない。

## できるようにしたいこと

- 装備を検索してステータスのシミュレーションができる
- ステータス計算について
  - メイン装備、サブ装備のステータスがシミュレーションできる。
  - セット効果が自動で反映される。
  - アバター効果が反映される。
  - アバティアの特殊効果が反映される。
  - 石板効果が反映される。
  - ロイヤル効果が反映される。
  - 冒険者の素のステータスが反映される。
  - 特殊コアの効果が反映される。
  - 強化コアの効果が反映される。

- ドーピングについて
  - ビタ
  - ドピ
  - ヌシステ
  - 羽
  - 魔獣缶
  - 増加ステータスの保存（ドピ後に装備変更したい）

- 保存機能
  - マネキン（装備）
  - マネキン（アバター）
  - マネキン（石板）
  - 装備の強化値
  - 特殊コア

- マスターデータの外部化
  - 装備一覧
  - セット効果一覧
  - アバター一覧
  - 特殊コア一覧
  - 強化レベル

## 開発

```bash
my-electron-app/
│
├── public/                  # 静的ファイル用ディレクトリ
│   ├── index.html           # メインHTMLファイル
│   └── icons/               # アイコンなどのリソース
├── resources/               # パッケージングされるリソース
│   └── data/                # データファイルを格納するディレクトリ
│       └── equipment.json
├── src/                     # ソースコードディレクトリ
│   ├── types/               # 型定義のためのディレクトリ
│   │   ├── global.d.ts      # グローバルで使用する型定義
│   │   └── models.d.ts      # モデルの型定義
│   ├── main/                # Electronのメインプロセススクリプト
│   │   └── main.ts          # メインプロセスのエントリポイント
│   │
│   ├── renderer/            # レンダラープロセススクリプト（Reactコンポーネント）
│   │   ├── components/      # 再利用可能なコンポーネント
│   │   │   ├── ...
│   │   │   └── ...
│   │   ├── hooks/           # カスタムフック
│   │   │   ├── ...
│   │   │   └── ...
│   │   ├── renderer.tsx     # Reactアプリのルートコンポーネント
│   │   └── ...
│   │
│   └── common/              # メインとレンダラー間で共有される可能性のあるユーティリティ
│       └── utils.ts
│
├── dist/                    # ビルドされたファイルが置かれるディレクトリ
│
├── node_modules/            # npmパッケージ
│
├── package.json             # プロジェクトのメタデータと依存関係
├── tsconfig.json            # TypeScriptのコンパイラ設定
├── webpack.config.js        # Webpackの設定
└── README.md                # プロジェクトの説明ファイル
```

- public/:
  - 静的ファイル、アイコン、HTMLファイルなど、Webサーバーから直接ロードされるファイルを格納します。
- src/:
  - アプリケーションのソースコード。main/ と renderer/ に分けて、Electronのメインプロセスとレンダラープロセスのコードを区別します。
- src/main/:
  - Electronのメインプロセスに関連するスクリプト。ウィンドウの作成、アプリケーションのライフサイクル管理などを担います。
- src/renderer/:
  - ReactによるUIの部分。components/ にはUIコンポーネント、hooks/ にはReactのカスタムフックを置きます。
- src/common/:
  - メインプロセスとレンダラープロセスで共用する可能性のあるコード（ユーティリティ関数など）を配置します。
- dist/:
  - Webpackなどのビルドツールによって生成された最終的な実行可能ファイルを格納します。
